title: "Streaming Migration Guide: Universal Streaming to Universal-3 Pro Streaming"

This guide walks through the process of upgrading from Universal Streaming to Universal-3 Pro Streaming for real-time audio transcription.

Universal-3 Pro Streaming delivers the highest accuracy with sub-300ms latency, native multilingual code switching, exceptional entity accuracy, and features like prompting and mid-stream configuration updates. It is optimized for **voice agents**, **agent assist**, and **all streaming use cases**.

### Get Started

Before we begin, make sure you have an AssemblyAI account and an API key. You can [sign up](https://assemblyai.com/dashboard/signup) for a free account and get your API key from your [dashboard](https://www.assemblyai.com/app/api-keys).

<Tip title="Quick upgrade">
  If you're already using Universal Streaming, you can quickly test
  Universal-3 Pro Streaming by switching the `speech_model` parameter to
  `"u3-rt-pro"` in your connection parameters. No other code changes are
  required — just update the model and start streaming.
</Tip>

## Key Differences

| Feature | Universal Streaming | Universal-3 Pro Streaming |
| --- | --- | --- |
| **`speech_model`** | `universal-streaming-english` or `universal-streaming-multilingual` | `u3-rt-pro` |
| **Turn detection** | Confidence-based (`end_of_turn_confidence_threshold`) | Punctuation-based (`min_turn_silence` + terminal punctuation) |
| **`min_turn_silence` default** | `400` ms | `100` ms |
| **`max_turn_silence` default** | `1280` ms | `1000` ms |
| **Formatting** | Opt-in via `format_turns=True` | Always on — formatting is built into the turn detection system |
| **`end_of_turn` and `turn_is_formatted`** | Can differ (`turn_is_formatted` arrives as a separate message after `end_of_turn`) | Always the same value — only one end-of-turn transcript per turn, always formatted |
| **Partials behavior** | Emitted for every subword as audio is processed | Emitted only during silence periods (at most one partial per silence period) |
| **`end_of_turn_confidence_threshold`** | Controls turn detection (default `0.4`) | Has no impact — turns are decided by punctuation |
| **`utterance` field** | Present in Turn messages for eager LLM inference | Not present |
| **Prompting** | Not supported | Supported via `prompt` connection parameter |
| **Keyterms prompting** | Supported | Supported |
| **UpdateConfiguration** | Not supported | Supported — update `keyterms_prompt`, `prompt`, `min_turn_silence`, `max_turn_silence` mid-stream |
| **ForceEndpoint** | Not supported | Supported — force the current turn to end immediately |
| **Multilingual support** | Separate models (`universal-streaming-english` vs `universal-streaming-multilingual`) | Single model with native multilingual code switching |
| **Entity accuracy** | Standard | Exceptional — credit cards, phone numbers, emails, addresses, names |

## Side-By-Side Code Comparison

Below is a side-by-side comparison of a basic snippet to transcribe live audio using Universal Streaming and Universal-3 Pro Streaming with a microphone:

<Tabs groupId="language">
    <Tab language="universal" title="Universal Streaming">

    ```python
    import pyaudio
    import websocket
    import json
    import threading
    import time
    from urllib.parse import urlencode
    from datetime import datetime

    YOUR_API_KEY = "YOUR-API-KEY"

    CONNECTION_PARAMS = {
        "sample_rate": 16000,
        "format_turns": True,
    }
    API_ENDPOINT_BASE_URL = "wss://streaming.assemblyai.com/v3/ws"
    API_ENDPOINT = f"{API_ENDPOINT_BASE_URL}?{urlencode(CONNECTION_PARAMS)}"

    FRAMES_PER_BUFFER = 800
    SAMPLE_RATE = CONNECTION_PARAMS["sample_rate"]
    CHANNELS = 1
    FORMAT = pyaudio.paInt16

    audio = None
    stream = None
    ws_app = None
    audio_thread = None
    stop_event = threading.Event()

    def on_open(ws):
        print("WebSocket connection opened.")
        def stream_audio():
            global stream
            while not stop_event.is_set():
                try:
                    audio_data = stream.read(FRAMES_PER_BUFFER, exception_on_overflow=False)
                    ws.send(audio_data, websocket.ABNF.OPCODE_BINARY)
                except Exception as e:
                    print(f"Error streaming audio: {e}")
                    break

        global audio_thread
        audio_thread = threading.Thread(target=stream_audio)
        audio_thread.daemon = True
        audio_thread.start()

    def on_message(ws, message):
        try:
            data = json.loads(message)
            msg_type = data.get('type')

            if msg_type == "Begin":
                session_id = data.get('id')
                expires_at = data.get('expires_at')
                print(f"\nSession began: ID={session_id}, ExpiresAt={datetime.fromtimestamp(expires_at)}")
            elif msg_type == "Turn":
                transcript = data.get('transcript', '')
                formatted = data.get('turn_is_formatted', False)

                if formatted:
                    print('\r' + ' ' * 80 + '\r', end='')
                    print(transcript)
                else:
                    print(f"\r{transcript}", end='')
            elif msg_type == "Termination":
                audio_duration = data.get('audio_duration_seconds', 0)
                print(f"\nSession terminated: {audio_duration}s of audio")
        except Exception as e:
            print(f"Error handling message: {e}")

    def on_error(ws, error):
        print(f"\nWebSocket Error: {error}")
        stop_event.set()

    def on_close(ws, close_status_code, close_msg):
        print(f"\nWebSocket Disconnected: Status={close_status_code}")
        global stream, audio
        stop_event.set()
        if stream:
            if stream.is_active():
                stream.stop_stream()
            stream.close()
        if audio:
            audio.terminate()

    def run():
        global audio, stream, ws_app

        audio = pyaudio.PyAudio()
        stream = audio.open(
            input=True,
            frames_per_buffer=FRAMES_PER_BUFFER,
            channels=CHANNELS,
            format=FORMAT,
            rate=SAMPLE_RATE,
        )
        print("Speak into your microphone. Press Ctrl+C to stop.")

        ws_app = websocket.WebSocketApp(
            API_ENDPOINT,
            header={"Authorization": YOUR_API_KEY},
            on_open=on_open,
            on_message=on_message,
            on_error=on_error,
            on_close=on_close,
        )

        ws_thread = threading.Thread(target=ws_app.run_forever)
        ws_thread.daemon = True
        ws_thread.start()

        try:
            while ws_thread.is_alive():
                time.sleep(0.1)
        except KeyboardInterrupt:
            print("\nStopping...")
            stop_event.set()
            if ws_app and ws_app.sock and ws_app.sock.connected:
                ws_app.send(json.dumps({"type": "Terminate"}))
                time.sleep(2)
            if ws_app:
                ws_app.close()
            ws_thread.join(timeout=2.0)

    if __name__ == "__main__":
        run()
    ```

Key changes when upgrading:

- **`speech_model`** — Set to `"u3-rt-pro"` instead of `"universal-streaming-english"` or `"universal-streaming-multilingual"`. Universal-3 Pro handles all languages natively with a single model.

- **`format_turns` is no longer needed** — Universal-3 Pro always returns formatted transcripts. Formatting is built into the turn detection system, so there's no need to opt in.

- **`end_of_turn_confidence_threshold` has no effect** — Universal-3 Pro uses punctuation-based turn detection instead of confidence-based. Remove this parameter if you were using it.

- **New parameters available** — You can now use `prompt` for custom prompting and `keyterms_prompt` for boosting recognition of specific terms at connection time.

## Turn Detection (Confidence-Based to Punctuation-Based)

One of the most significant differences between Universal Streaming and Universal-3 Pro Streaming is how turn detection works.

### Universal Streaming: Confidence-based

Universal Streaming uses a confidence-based system with three parameters:

| Parameter | Default | Description |
| --- | --- | --- |
| `end_of_turn_confidence_threshold` | `0.4` | Confidence threshold (0.0–1.0) to trigger end of turn |
| `min_turn_silence` | `400` ms | Minimum silence required before checking confidence |
| `max_turn_silence` | `1280` ms | Maximum silence before forcing end of turn |

The model evaluates an `end_of_turn_confidence` score during silence. If the score exceeds `end_of_turn_confidence_threshold` after `min_turn_silence`, the turn ends. Otherwise, the turn is forced to end after `max_turn_silence`.

### Universal-3 Pro Streaming: Punctuation-based

Universal-3 Pro uses a punctuation-based system controlled by two parameters:

| Parameter | Default | Description |
| --- | --- | --- |
| `min_turn_silence` | `100` ms | Silence before a speculative end-of-turn (EOT) check fires |
| `max_turn_silence` | `1000` ms | Maximum silence before a turn is forced to end |

When silence reaches `min_turn_silence`, the model transcribes the audio and checks for terminal punctuation (`.` `?` `!`):

- **Terminal punctuation found** — the turn ends (`end_of_turn: true`).
- **No terminal punctuation** — a partial is emitted (`end_of_turn: false`) and the turn continues.
- If silence reaches `max_turn_silence`, the turn is forced to end regardless of punctuation.

<Note>
Because turn detection is now punctuation-based, the `end_of_turn_confidence_threshold` parameter has no impact in Universal-3 Pro. You can safely remove it from your connection parameters.
</Note>

Key differences in message handling:

- **`turn_is_formatted` and `end_of_turn` are always the same** — In Universal Streaming, the formatted transcript arrives as a separate message after the end-of-turn message (you check `turn_is_formatted`). In Universal-3 Pro, there is only one end-of-turn transcript per turn and it is always formatted. You can simply check `end_of_turn` to detect a formatted, final transcript.

- **No `utterance` field** — Universal Streaming includes an `utterance` field for eager LLM inference. Universal-3 Pro does not include this field. Instead, use partials (`end_of_turn: false`) for eager inference.

- **No `end_of_turn_confidence` to evaluate** — Universal Streaming Turn messages include an `end_of_turn_confidence` score. While this field may still be present in Universal-3 Pro, turn decisions are made by punctuation, not by this score.

## Partials Behavior

Partials are `Turn` events where `end_of_turn` is `false`.

**Universal Streaming** emits partials continuously as audio is processed, including updates for every subword. This produces a stream of incremental transcript updates.

**Universal-3 Pro Streaming** emits partials only during periods of silence. When `min_turn_silence` is reached, the model checks for terminal punctuation. If no terminal punctuation is found, a partial is emitted and the turn continues. There is at most one partial per silence period.

This means Universal-3 Pro produces fewer, more meaningful partials. If you were relying on frequent subword-level partials, note this behavioral change. For eager LLM inference on partials, we recommend setting `min_turn_silence` to `100` (the default).

<Warning title="Entity splitting trade-off">
  Setting `min_turn_silence` too low can split entities like phone numbers and
  emails across partials. LLM steps typically fix this for voice agents, but
  test carefully with your use case.
</Warning>

## New Universal-3 Pro Features

Universal-3 Pro Streaming introduces several features not available in Universal Streaming.

### Prompting

Universal-3 Pro supports custom prompts to improve transcription accuracy. A [default prompt](/docs/streaming/prompting#start-with-the-default-prompt) optimized for turn detection is automatically applied. You can provide your own prompt via the `prompt` connection parameter.

```python
CONNECTION_PARAMS = {
    "sample_rate": 16000,
    "speech_model": "u3-rt-pro",
    "prompt": "Transcribe a customer support call about software billing.",
}
```

For detailed guidance on crafting effective prompts, see the [Prompting Guide (Streaming)](/docs/streaming/prompting).

### Keyterms Prompting

Boost recognition of specific terms using `keyterms_prompt`:

```python
CONNECTION_PARAMS = {
    "sample_rate": 16000,
    "speech_model": "u3-rt-pro",
    "keyterms_prompt": ["AssemblyAI", "Universal-3 Pro"],
}
```

<Note>
  Keyterms longer than 50 characters are ignored. Requests with more than 100 keyterms will result in an error. See [Keyterms prompting](/docs/streaming/keyterms-prompting) for details.
</Note>

### UpdateConfiguration

You can update configuration during an active session without reconnecting. This is especially useful for dynamically adjusting keyterms as your voice agent conversation progresses through different stages.

```python
# Update keyterms for a new conversation stage
ws.send(json.dumps({
    "type": "UpdateConfiguration",
    "keyterms_prompt": ["account number", "routing number"],
}))

# Update multiple parameters at once
ws.send(json.dumps({
    "type": "UpdateConfiguration",
    "keyterms_prompt": ["cardiology", "echocardiogram"],
    "max_turn_silence": 5000,
    "min_turn_silence": 200,
}))

# Clear keyterms
ws.send(json.dumps({
    "type": "UpdateConfiguration",
    "keyterms_prompt": [],
}))
```

You can update `keyterms_prompt`, `prompt`, `max_turn_silence`, and `min_turn_silence` mid-stream.

### ForceEndpoint

Force the current turn to end immediately by sending a `ForceEndpoint` message:

```python
ws.send(json.dumps({"type": "ForceEndpoint"}))
```

This is useful when your application knows the user has finished speaking based on external signals (e.g., a button press or VAD).

## Language Support

**Universal Streaming** requires choosing between two separate models:
- `universal-streaming-english` — English only (default)
- `universal-streaming-multilingual` — English, Spanish, French, German, Italian, and Portuguese

**Universal-3 Pro Streaming** uses a single model (`u3-rt-pro`) with native multilingual code switching. You no longer need to select a language-specific model. The model automatically handles multiple languages within the same session.

To improve accuracy for a specific language, you can provide language information in the prompt:

```python
CONNECTION_PARAMS = {
    "sample_rate": 16000,
    "speech_model": "u3-rt-pro",
    "prompt": "Transcribe Spanish. Transcribe verbatim. Rules:\n    Always include punctuation in output.\n    Use period/question mark ONLY for complete sentences.\n    Use comma for mid-sentence pauses.\n    Use no punctuation for incomplete trailing speech.\n    Filler words (um, uh, so, like) indicate speaker will continue.",
}
```

For multilingual conversations, use: `"Transcribe multilingual conversation in English, Spanish, and German."`

For the full language list and dialect reference, see [Supported languages](/docs/streaming/universal-3-pro/supported-languages).
