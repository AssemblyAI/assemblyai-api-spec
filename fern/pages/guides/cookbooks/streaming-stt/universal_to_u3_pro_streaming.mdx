title: "Streaming Migration Guide: Universal Streaming to Universal-3 Pro Streaming"

This guide walks through the process of upgrading from Universal Streaming to Universal-3 Pro Streaming for real-time audio transcription.

Universal-3 Pro Streaming delivers the highest accuracy with sub-300ms latency, native multilingual code switching, exceptional entity accuracy, and features like prompting and mid-stream configuration updates. It is optimized for **voice agents**, **agent assist**, and **all streaming use cases**.

### Get Started

Before we begin, make sure you have an AssemblyAI account and an API key. You can [sign up](https://assemblyai.com/dashboard/signup) for a free account and get your API key from your [dashboard](https://www.assemblyai.com/app/api-keys).

<Tip title="Quick upgrade">
  If you're already using Universal Streaming, you can quickly test Universal-3
  Pro Streaming by switching the `speech_model` parameter to `"u3-rt-pro"` in
  your connection parameters. No other code changes are required — just update
  the model and start streaming.
</Tip>

## Why upgrade

- **Exceptional entity accuracy** — credit card numbers, phone numbers, email addresses, physical addresses, and names captured correctly at streaming speed
- **Promptable model** — feed domain vocabulary (up to 1,000 keyterms) and formatting instructions, and the model applies them as audio arrives
- **Better turn detection** — punctuation-based system that waits when customers pause mid-thought and responds when they're done, fixing the cut-off conversation problem
- **Native multilingual code-switching** — English, Spanish, German, French, Portuguese, Italian handled natively in a single model
- **Sub-300ms latency**
- **Mid-stream configuration** — update keyterms, prompts, and silence parameters without dropping the connection

## Key Differences

| Feature                                   | Universal Streaming                                                                   | Universal-3 Pro Streaming                                                          |
| ----------------------------------------- | ------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------- |
| **`speech_model`**                        | `universal-streaming-english` or `universal-streaming-multilingual`                   | `u3-rt-pro`                                                                        |
| **Turn detection**                        | Confidence-based (`end_of_turn_confidence_threshold`)                                 | Punctuation-based (`min_turn_silence` + terminal punctuation)                      |
| **`min_turn_silence` default**            | `400` ms                                                                              | `100` ms                                                                           |
| **`max_turn_silence` default**            | `1280` ms                                                                             | `1000` ms                                                                          |
| **Formatting**                            | Opt-in via `format_turns=True`                                                        | Always on — formatting is built into the turn detection system                     |
| **`end_of_turn` and `turn_is_formatted`** | Can differ (`turn_is_formatted` arrives as a separate message after `end_of_turn`)    | Always the same value — only one end-of-turn transcript per turn, always formatted |
| **Partials behavior**                     | Emitted for every subword as audio is processed                                       | Emitted only during silence periods (at most one partial per silence period)       |
| **Prompting**                             | Not supported                                                                         | Supported via `prompt` connection parameter                                        |
| **Keyterms prompting**                    | Supported                                                                             | Supported                                                                          |
| **`vad_threshold` default**               | `0.4`                                                                                 | `0.3` — more sensitive; may need tuning in noisy environments                      |
| **Multilingual support**                  | Separate models (`universal-streaming-english` vs `universal-streaming-multilingual`) | Single model with native multilingual code switching                               |

For a complete working example, see the [Universal-3 Pro Streaming quickstart](/docs/streaming/universal-3-pro#quickstart).

## Connection Parameters

<Tabs groupId="language">
    <Tab language="universal" title="Universal Streaming">
        ```python
        CONNECTION_PARAMS = {
            "sample_rate": 16000,
            "format_turns": True,  # Opt-in to get formatted transcripts
        }
        API_ENDPOINT_BASE_URL = "wss://streaming.assemblyai.com/v3/ws"
        API_ENDPOINT = f"{API_ENDPOINT_BASE_URL}?{urlencode(CONNECTION_PARAMS)}"
        ```
    </Tab>

    <Tab language="u3pro" title="Universal-3 Pro Streaming">
        ```python
        CONNECTION_PARAMS = {
            "sample_rate": 16000,
            "speech_model": "u3-rt-pro",  # Select Universal-3 Pro model
        }
        API_ENDPOINT_BASE_URL = "wss://streaming.assemblyai.com/v3/ws"
        API_ENDPOINT = f"{API_ENDPOINT_BASE_URL}?{urlencode(CONNECTION_PARAMS)}"
        ```
    </Tab>

</Tabs>

Key changes when upgrading:

- **`speech_model`** — Set to `"u3-rt-pro"` instead of `"universal-streaming-english"` or `"universal-streaming-multilingual"`. Universal-3 Pro handles all languages natively with a single model.

- **`format_turns` is no longer needed** — Universal-3 Pro always returns formatted transcripts. Formatting is built into the turn detection system, so there's no need to opt in.

- **`end_of_turn_confidence_threshold` has no effect** — Universal-3 Pro uses punctuation-based turn detection instead of confidence-based. Remove this parameter if you were using it.

- **New parameters available** — You can now use `prompt` for custom prompting and `keyterms_prompt` for boosting recognition of specific terms at connection time.

For details on configuring turn detection, partials behavior, prompting, keyterms prompting, mid-stream configuration updates, and forcing turn endpoints, see [Universal-3 Pro Streaming](/docs/streaming/universal-3-pro).

## Language support

Universal-3 Pro uses a single model (`u3-rt-pro`) with native multilingual code-switching for English, Spanish, German, French, Portuguese, and Italian. See [Supported languages](/docs/streaming/universal-3-pro/supported-languages) for the full list.
