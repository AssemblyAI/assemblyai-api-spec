title: "Transcribe Multiple Files Simultaneously Using the Python SDK"

In this guide, we'll show you how to use the AssemblyAI API to transcribe multiple audio files at once. This guide focuses on demonstrating how to use the AssemblyAI Python SDK to achieve this.

## Quickstart

```python
import assemblyai as aai
import os
from concurrent.futures import ThreadPoolExecutor, as_completed

aai.settings.api_key = "YOUR_API_KEY"
batch_folder = "audio"
transcription_result_folder = "transcripts"

def transcribe_audio(audio_file):
    transcriber = aai.Transcriber()
    transcript = transcriber.transcribe(os.path.join(batch_folder, audio_file))
    if transcript.status == aai.TranscriptStatus.completed:
        with open(f"{transcription_result_folder}/{audio_file}.txt", "w") as f:
            f.write(transcript.text)
        return audio_file, "completed"
    else:
        print(f"Error transcribing {audio_file}: {transcript.error}")
        return audio_file, "error"

max_workers = 50  # Adjust based on your concurrency limit
files = os.listdir(batch_folder)

with ThreadPoolExecutor(max_workers=max_workers) as executor:
    futures = {
        executor.submit(transcribe_audio, f): f for f in files
    }
    for future in as_completed(futures):
        filename, status = future.result()
        print(f"{filename}: {status}")

print("All transcriptions are complete.")
```

<Note title="Concurrency and rate limits">

The AssemblyAI API has a concurrency limit (default 200 for paid accounts) and a global rate limit of 20,000 requests per five minutes. The SDK handles queuing automatically — if you exceed your concurrency limit, jobs are queued server-side. However, if you're submitting thousands of files, consider limiting the number of concurrent threads to avoid hitting the API rate limit. For more details, see [Scaling async transcription](/docs/deployment/account-management#scaling-async-transcription).

</Note>

## Step-by-Step Guide

Install the SDK.

```python
pip install -U assemblyai
```

Import the `assemblyai` package and set the API key. Import `os` and `concurrent.futures` for file path interactions and managed concurrent task processing respectively.

```python
import assemblyai as aai
import os
from concurrent.futures import ThreadPoolExecutor, as_completed

aai.settings.api_key = "YOUR_API_KEY"
```

Set the folders. The `batch` folder contains the audio files that you want to process and transcribe. The `transcription_result_folder` stores the .txt transcript files.

```python
batch_folder = "audio"
transcription_result_folder = "transcripts"
```

Create a `Transcriber` object.

```python
transcriber = aai.Transcriber()
```

Function to transcribe an audio file. Once the transcript is complete, a .txt file is generated to the `transcription_result_folder`. If there is an error with the transcription, it will not be processed to the results folder.

```python
def transcribe_audio(audio_file):
    transcriber = aai.Transcriber()
    transcript = transcriber.transcribe(os.path.join(batch_folder, audio_file))
    if transcript.status == aai.TranscriptStatus.completed:
        with open(f"{transcription_result_folder}/{audio_file}.txt", "w") as f:
            f.write(transcript.text)
        return audio_file, "completed"
    else:
        print(f"Error transcribing {audio_file}: {transcript.error}")
        return audio_file, "error"
```

Use a `ThreadPoolExecutor` to transcribe files concurrently with a controlled number of workers. The `max_workers` parameter limits how many files are submitted in parallel — adjust this based on your [concurrency limit](/docs/deployment/account-management#pre-recorded-audio-speech-to-text-usage-limits). Once all tasks complete, you'll see the "All transcriptions are complete" message.

```python
max_workers = 50  # Adjust based on your concurrency limit
files = os.listdir(batch_folder)

with ThreadPoolExecutor(max_workers=max_workers) as executor:
    futures = {
        executor.submit(transcribe_audio, f): f for f in files
    }
    for future in as_completed(futures):
        filename, status = future.result()
        print(f"{filename}: {status}")

print("All transcriptions are complete.")
```

<Tip title="Scaling tips">

- The AssemblyAI API queues jobs that exceed your concurrency limit, so you can safely submit more files than your limit allows.
- If you're processing thousands of files, limit `max_workers` to avoid hitting the global API rate limit of 20,000 requests per five minutes.
- For high-volume workloads, consider using [webhooks](/docs/getting-started/webhooks) instead of polling to reduce API calls.

</Tip>
