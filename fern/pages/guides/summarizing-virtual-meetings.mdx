In this guide, we'll show you how to use [LLM Gateway](/docs/llm-gateway/overview) to generate summaries of your virtual meetings. You'll transcribe your audio and then use LLM Gateway to produce a custom summary.

<Note>
  The `summarization`, `summary_model`, and `summary_type` parameters on the transcription API are deprecated. Use LLM Gateway as shown below for more flexible and powerful summaries.
</Note>

## Get started

Before we begin, make sure you have an AssemblyAI account and an API key. You can [sign up](https://assemblyai.com/dashboard/signup) for a free account and get your API key from your dashboard.

```python
    "speech_models": ["universal-3-pro", "universal-2"],
    "language_detection": True

transcript_id = response.json()['id']

while True:
    transcription_result = requests.get(polling_endpoint, headers=headers).json()
    if transcription_result['status'] == 'completed':

    elif transcription_result['status'] == 'error':
        raise RuntimeError(f"Transcription failed: {transcription_result['error']}")
```

```python
prompt = """Provide a summary of this meeting transcript in bullet point format.

llm_gateway_data = {

    "messages": [
        {"role": "user", "content": f"{prompt}\n\nTranscript: {transcription_result['text']}"}

}

response = requests.post(
    "https://llm-gateway.assemblyai.com/v1/chat/completions",
    headers=headers,
    json=llm_gateway_data
)

result = response.json()["choices"][0]["message"]["content"]
print(result)
```

## Understanding the response

The LLM Gateway returns the summary in the `choices[0].message.content` field. You can customize the format and style of the summary by adjusting the prompt.

## Best practices

- Customize the prompt to match your meeting type (e.g., standup, all-hands, client call).
- For longer meetings, consider breaking the transcript into sections and summarizing each one separately.
- Use [Structured Outputs](/docs/llm-gateway/structured-outputs) if you need the summary in a specific JSON format.
- Try different [LLM models](/docs/llm-gateway/overview#available-models) to find the one that produces the best summaries for your use case.

## Advanced usage

You can define custom summary formats by adjusting the prompt:

```python
prompt = """Summarize this meeting transcript using the following format:
## Meeting Summary
- **Key Discussion Points**: [list main topics discussed]
- **Decisions Made**: [list any decisions]
- **Action Items**: [list action items with owners if mentioned]
- **Next Steps**: [list next steps]"""
```
