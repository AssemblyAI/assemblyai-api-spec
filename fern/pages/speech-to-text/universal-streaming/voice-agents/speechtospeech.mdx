AssemblyAI's Speech-to-Speech (S2S) API is a native real-time voice conversation endpoint. Unlike a traditional STT → LLM → TTS pipeline, the S2S API handles everything in a single WebSocket connection: it listens to the user, understands what they said, generates a response, and speaks it back — all with low latency.

**Key capabilities:**

- **Built-in VAD** — server-side voice activity detection, no client configuration needed
- **Automatic turn detection** — the model decides when the user has finished speaking
- **Native audio output** — streams PCM16 audio back directly, no separate TTS step
- **Function calling** — register tools and handle `function.call` / `function.result` events
- **Barge-in / interruption** — users can interrupt the agent mid-response

## Quickstart

Connect via WebSocket and start a conversation in under a minute.

<Tab title="Python">
```python
import asyncio
import base64
import json
import sounddevice as sd
import numpy as np
import websockets

API_KEY = "YOUR_API_KEY"
URL = "wss://speech-to-speech.us.assemblyai.com/v1/realtime"

SAMPLE_RATE = 24000
CHANNELS = 1
DTYPE = "int16"

async def main():
    headers = {"Authorization": f"Bearer {API_KEY}"}

    async with websockets.connect(URL, additional_headers=headers) as ws:
        print("Connected. Waiting for session...")

        # Start mic capture in background
        loop = asyncio.get_running_loop()
        audio_queue = asyncio.Queue()

        def mic_callback(indata, frames, time, status):
            loop.call_soon_threadsafe(audio_queue.put_nowait, bytes(indata))

        async def send_audio():
            while True:
                chunk = await audio_queue.get()
                b64 = base64.b64encode(chunk).decode()
                await ws.send(json.dumps({"type": "audio.append", "audio": b64}))

        send_task = asyncio.create_task(send_audio())

        with sd.InputStream(samplerate=SAMPLE_RATE, channels=CHANNELS,
                            dtype=DTYPE, callback=mic_callback), \
             sd.OutputStream(samplerate=SAMPLE_RATE, channels=CHANNELS,
                             dtype=DTYPE) as speaker:

            async for message in ws:
                event = json.loads(message)
                t = event.get("type")

                if t == "session.ready":
                    print("Session ready — start speaking")

                elif t == "transcript.user":
                    print(f"You: {event['text']}")

                elif t == "response.audio":
                    audio = base64.b64decode(event["data"])
                    speaker.write(np.frombuffer(audio, dtype=np.int16))

                elif t == "response.transcript":
                    print(f"Agent: {event['text']}")

                elif t == "error":
                    print(f"Error: {event.get('message')}")
                    break

asyncio.run(main())
```

```javascript
import WebSocket from "ws";
import mic from "mic";
import Speaker from "speaker";

const API_KEY = "YOUR_API_KEY";
const URL = "wss://speech-to-speech.us.assemblyai.com/v1/realtime";

const SAMPLE_RATE = 24000;

const ws = new WebSocket(URL, {
  headers: { Authorization: `Bearer ${API_KEY}` },
});

const speaker = new Speaker({
  channels: 1,
  bitDepth: 16,
  sampleRate: SAMPLE_RATE,
});

const microphone = mic({
  rate: SAMPLE_RATE,
  channels: 1,
  bitwidth: 16,
  encoding: "signed-integer",
});

const micStream = microphone.getAudioStream();

ws.on("open", () => {
  console.log("Connected. Waiting for session...");
});

ws.on("message", (data) => {
  const event = JSON.parse(data.toString());

  switch (event.type) {
    case "session.ready":
      console.log("Session ready — start speaking");
      microphone.start();
      break;

    case "transcript.user":
      console.log(`You: ${event.text}`);
      break;

    case "response.audio":
      speaker.write(Buffer.from(event.data, "base64"));
      break;

    case "response.transcript":
      console.log(`Agent: ${event.text}`);
      break;

    case "error":
      console.error(`Error: ${event.message}`);
      ws.close();
      break;
  }
});

micStream.on("data", (chunk) => {
  if (ws.readyState === WebSocket.OPEN) {
    ws.send(
      JSON.stringify({
        type: "audio.append",
        audio: chunk.toString("base64"),
      })
    );
  }
});
```

## Audio format

Both input and output audio use the same format:

| Property      | Value                                    |
| ------------- | ---------------------------------------- |
| Encoding      | PCM16 (16-bit signed integer, little-endian) |
| Sample rate   | 24,000 Hz                                |
| Channels      | Mono                                     |

Send audio as base64-encoded PCM16 chunks. Chunk size doesn't matter — the server buffers and processes continuously.

## Events reference

### Client → Server

#### `audio.append`

Stream PCM16 audio to the agent. Send this continuously as the user speaks.

```json
{
  "type": "audio.append",
  "audio": "<base64-encoded PCM16>"
}
```

| Field   | Type   | Description                       |
| ------- | ------ | --------------------------------- |
| `audio` | string | Base64-encoded PCM16 mono 24kHz audio |

---

#### `session.configure`

Configure the session after `session.ready`. Can be sent multiple times to update the system prompt or tools mid-conversation.

```json
{
  "type": "session.configure",
  "session": {
    "system_prompt": "You are a helpful assistant.",
    "tools": []
  }
}
```

| Field                    | Type   | Description                              |
| ------------------------ | ------ | ---------------------------------------- |
| `session.system_prompt`  | string | Sets the agent's personality and context |
| `session.tools`          | array  | Tool definitions (see [Function calling](#function-calling)) |

---

#### `response.create`

Manually trigger a response from the agent. Useful when VAD is disabled or you want to force a turn.

```json
{ "type": "response.create" }
```

---

#### `response.cancel`

Cancel the agent's current response (stop it from speaking). Use this to implement barge-in.

```json
{
  "type": "response.cancel",
  "response_id": "resp_abc123"
}
```

| Field         | Type   | Description                                  |
| ------------- | ------ | -------------------------------------------- |
| `response_id` | string | The ID of the response to cancel, from `response.started` |

---

#### `function.result`

Send the result of a function call back to the agent.

```json
{
  "type": "function.result",
  "call_id": "call_abc123",
  "result": "72°F and sunny in Tokyo."
}
```

| Field     | Type   | Description                                |
| --------- | ------ | ------------------------------------------ |
| `call_id` | string | The `call_id` from the `function.call` event |
| `result`  | string | The result to return to the agent          |

---

### Server → Client

#### `session.ready`

Sent once the WebSocket connection is established and the session is ready to receive audio.

```json
{
  "type": "session.ready"
}
```

---

#### `speech.started`

Sent when the server detects the user has started speaking.

```json
{
  "type": "speech.started",
  "timestamp": 1234567890.123
}
```

---

#### `speech.stopped`

Sent when the server detects the user has stopped speaking.

```json
{
  "type": "speech.stopped",
  "timestamp": 1234567890.456
}
```

---

#### `transcript.user.delta`

Partial (interim) transcript of what the user is currently saying. Updates as the user speaks.

```json
{
  "type": "transcript.user.delta",
  "text": "What's the weather in"
}
```

---

#### `transcript.user`

Final transcript of the user's utterance, sent after `speech.stopped`.

```json
{
  "type": "transcript.user",
  "text": "What's the weather in Tokyo?",
  "item_id": "item_abc123"
}
```

---

#### `response.started`

Sent when the agent begins generating a response.

```json
{
  "type": "response.started",
  "response_id": "resp_abc123"
}
```

---

#### `response.audio`

A chunk of the agent's spoken response as base64 PCM16 audio. Arrives in real-time as the agent speaks — decode and play immediately for lowest latency.

```json
{
  "type": "response.audio",
  "data": "<base64-encoded PCM16>"
}
```

---

#### `response.transcript`

The full text of the agent's response, sent after the response completes.

```json
{
  "type": "response.transcript",
  "text": "It's currently 72°F and sunny in Tokyo."
}
```

---

#### `response.done`

Sent when the agent finishes speaking.

```json
{
  "type": "response.done"
}
```

---

#### `response.interrupted`

Sent when the agent's response was interrupted (e.g. the user started speaking).

```json
{
  "type": "response.interrupted"
}
```

---

#### `function.call`

Sent when the agent wants to call a registered tool.

```json
{
  "type": "function.call",
  "call_id": "call_abc123",
  "name": "get_weather",
  "args": { "city": "Tokyo" }
}
```

| Field     | Type   | Description                          |
| --------- | ------ | ------------------------------------ |
| `call_id` | string | Unique ID — include this in `function.result` |
| `name`    | string | Name of the tool to call             |
| `args`    | object | Arguments parsed from the user's speech |

---

#### `error`

Sent on connection or session errors.

```json
{
  "type": "error",
  "message": "Invalid API key"
}
```

## Session configuration

### System prompt

Set the agent's personality, role, and behaviour. Can be updated mid-session.

```json
{
  "type": "session.configure",
  "session": {
    "system_prompt": "You are a friendly customer support agent for Acme Corp. Keep responses under 2 sentences. Never make up information — if you don't know, say so."
  }
}
```

**Tips for voice-first prompts:**

- Keep instructions concise — the model reads your prompt, not the user
- Use "Never say X" to ban specific phrases (e.g. "Never say 'Certainly' or 'Absolutely'")
- Instruct the agent to keep responses short: "Max 2 sentences per turn"
- For tool-based agents, tell the agent when and how to use each tool

## Function calling

Register tools to let the agent take real-world actions. The agent decides when to call a tool based on the conversation.

### Register tools

Include tool definitions in `session.configure`:

```json
{
  "type": "session.configure",
  "session": {
    "system_prompt": "You are a helpful assistant. Use get_weather to look up weather when asked.",
    "tools": [
      {
        "type": "function",
        "name": "get_weather",
        "description": "Get the current weather for a city.",
        "parameters": {
          "type": "object",
          "properties": {
            "city": {
              "type": "string",
              "description": "The city name"
            }
          },
          "required": ["city"]
        }
      }
    ]
  }
}
```

### Handle function calls

When the agent wants to call a tool, it emits a `function.call` event. Execute the function and send back the result:

### Tool schema

Tools use standard JSON Schema format. Supported parameter types: `string`, `number`, `boolean`, `array`, `object`.

```json
{
  "type": "function",
  "name": "tool_name",
  "description": "What this tool does — the model uses this to decide when to call it",
  "parameters": {
    "type": "object",
    "properties": {
      "param": {
        "type": "string",
        "description": "What this parameter is"
      },
      "category": {
        "type": "string",
        "enum": ["option_a", "option_b", "option_c"]
      }
    },
    "required": ["param"]
  }
}
```

## Interruptions

The server detects when the user starts speaking mid-response (`speech.started`) and will automatically cancel the current response. You can also cancel manually:

```python
# Cancel the current agent response
await ws.send(json.dumps({
    "type": "response.cancel",
    "response_id": current_response_id,  # from response.started
}))
```

Track the current `response_id` by saving it when `response.started` arrives, and clearing it on `response.done` or `response.interrupted`.

## Browser integration

Browsers cannot set the `Authorization` header on WebSocket connections. The recommended pattern is a lightweight server-side proxy that adds the auth header before forwarding to AssemblyAI.

```
Browser ──── ws:// ──── Your proxy ──── wss://speech-to-speech.us.assemblyai.com
(no key)                 (adds auth)
```

**Minimal Node.js proxy:**

```javascript
import http from "http";
import { WebSocket, WebSocketServer } from "ws";

const AAI_URL = "wss://speech-to-speech.us.assemblyai.com/v1/realtime";
const API_KEY = process.env.ASSEMBLYAI_API_KEY;

const server = http.createServer();
const wss = new WebSocketServer({ server });

wss.on("connection", (clientWs) => {
  const aaiWs = new WebSocket(AAI_URL, {
    headers: { Authorization: `Bearer ${API_KEY}` },
  });

  clientWs.on("message", (data) => {
    if (aaiWs.readyState === WebSocket.OPEN) aaiWs.send(data);
  });

  aaiWs.on("message", (data) => {
    if (clientWs.readyState === WebSocket.OPEN) clientWs.send(data);
  });

  aaiWs.on("close", (code, reason) => clientWs.close(code, reason));
  clientWs.on("close", () => aaiWs.close());
});

server.listen(8080);
```

The browser then connects to `ws://localhost:8080` (or your deployed proxy URL) with no authentication needed.

```python
# Register tools
async def get_weather(params):
    city = params.arguments.get("city")
```

```python
    args = event["args"]

    if name == "check_order_status":
        # Replace with real lookup
        result = f"Order {args['order_id']} is out for delivery and will arrive today."
```

## Event flow diagram

```
Client                              Server
  │                                   │
  │── WebSocket connect ──────────────►│
  │                                   │
  │◄─── session.ready ────────────────│
  │                                   │
  │── session.configure ─────────────►│  (set system prompt + tools)
  │                                   │
  │── audio.append (stream) ─────────►│  (user speaks)
  │── audio.append (stream) ─────────►│
  │── audio.append (stream) ─────────►│
  │                                   │
  │◄─── transcript.user.delta ────────│  (partial transcript)
  │◄─── transcript.user.delta ────────│
  │◄─── transcript.user ──────────────│  (final transcript)
  │                                   │
  │◄─── response.started ─────────────│
  │◄─── response.audio ───────────────│  (agent speaks)
  │◄─── response.audio ───────────────│
  │◄─── response.transcript ──────────│  (agent text)
  │◄─── response.done ────────────────│
  │                                   │
  │  [tool call example]              │
  │◄─── function.call ────────────────│
```
