## Overview

This guide covers integrating AssemblyAI's **Universal-3 Pro (u3-rt-pro)** streaming speech-to-text model into Pipecat voice agents. U3-Pro is AssemblyAI's next-generation model built specifically for voice agent use cases, featuring advanced turn detection, prompt engineering capabilities, speaker diarization, and optimized latency.

### Key features

- **Two-mode turn detection**: Choose between Pipecat-controlled (VAD + Smart Turn) or AssemblyAI's built-in turn detection (STT-based)
- **Dynamic parameter updates**: Change configuration mid-conversation without reconnection
- **Speaker diarization**: Identify and label different speakers in multi-party conversations
- **Custom prompting**: Guide transcription behavior with custom instructions
- **Keyterms boosting**: Improve recognition of specific words and names
- **Language detection**: Automatic language identification with confidence scoring

### Critical configuration requirements

- **Choose your turn detection mode**: Pipecat mode (default, recommended) or AssemblyAI's built-in turn detection (STT mode)
- **Set `min_turn_silence` to `100`** for best latency
- **Enable VAD in Pipecat mode** for interruption handling
- **Set `format_turns` to `True`** (no added latency with U3-Pro)

## Two-mode turn detection

Pipecat offers two distinct approaches to turn detection with U3-Pro.

### Pipecat mode (default, recommended)

**When to use:** Most voice agent applications requiring responsive interruptions.

```python
stt = AssemblyAISTTService(
    api_key=os.getenv("ASSEMBLYAI_API_KEY"),
    connection_params=AssemblyAIConnectionParams(
        speech_model="u3-rt-pro",
        min_turn_silence=100,
        format_turns=True,
    ),
    vad_force_turn_endpoint=True,  # Default — Pipecat mode
)
```

**How it works:**

- VAD + Smart Turn analyzer controls when the user is done speaking
- `ForceEndpoint` message sent to AssemblyAI on VAD silence detection
- `max_turn_silence` automatically synchronized with `min_turn_silence`
- Best for low-latency, responsive voice agents

### AssemblyAI's built-in turn detection (STT mode)

**When to use:** When you want AssemblyAI's built-in turn detection to control turn endings (experimental).

```python
stt = AssemblyAISTTService(
    api_key=os.getenv("ASSEMBLYAI_API_KEY"),
    connection_params=AssemblyAIConnectionParams(
        speech_model="u3-rt-pro",
        min_turn_silence=100,
        max_turn_silence=5000,  # Now respected
        format_turns=True,
    ),
    vad_force_turn_endpoint=False,  # AssemblyAI's built-in turn detection (STT mode)
)
```

**How it works:**

- AssemblyAI's built-in turn detection controls when the user is done speaking
- All timing parameters are respected as configured
- Emits `UserStartedSpeakingFrame` / `UserStoppedSpeakingFrame`
- Uses `SpeechStarted` events for fast barge-in
- Only available with `u3-rt-pro` (other models require Pipecat mode)

<Note>
  AssemblyAI's built-in turn detection uses the STT model's understanding of speech patterns to determine turn boundaries, rather than relying on local VAD silence detection.
</Note>

### Comparison: Pipecat mode vs AssemblyAI's built-in turn detection (STT mode)

| Feature                          | Pipecat Mode (default)                             | AssemblyAI's Built-in Turn Detection (STT Mode) |
| -------------------------------- | -------------------------------------------------- | ------------------------------------------------ |
| Turn detection control           | VAD + Smart Turn                                   | AssemblyAI's built-in model                      |
| ForceEndpoint sent               | Yes (on VAD stop)                                  | No                                               |
| `max_turn_silence` behavior      | Auto-synchronized with `min_silence`               | Respected as configured                          |
| UserStarted/StoppedFrame         | Not emitted from STT                               | Emitted from STT                                 |
| VAD requirement                  | Required (for interruptions)                       | Optional                                         |
| Recommended for                  | Most voice agent use cases                         | Experimental / specific timing requirements      |
| Model support                    | All models                                         | `u3-rt-pro` only                                 |
| Interruption latency             | Fast (local VAD)                                   | Fast (SpeechStarted events)                      |
| Configuration complexity         | Simpler (auto-sync)                                | More control (manual timing)                     |

## Basic example (Pipecat mode)

```python
import asyncio
import os
import sys

from dotenv import load_dotenv
from loguru import logger

from pipecat.audio.vad.silero import SileroVADAnalyzer
from pipecat.frames.frames import LLMRunFrame
from pipecat.pipeline.pipeline import Pipeline
from pipecat.pipeline.runner import PipelineRunner
from pipecat.pipeline.task import PipelineParams, PipelineTask
from pipecat.processors.aggregators.llm_context import LLMContext
from pipecat.processors.aggregators.llm_response_universal import (
    LLMContextAggregatorPair,
    LLMUserAggregatorParams,
)
from pipecat.services.assemblyai.models import AssemblyAIConnectionParams
from pipecat.services.assemblyai.stt import AssemblyAISTTService
from pipecat.services.rime.tts import RimeTTSService
from pipecat.services.cerebras.llm import CerebrasLLMService
from pipecat.transports.local.audio import LocalAudioTransport, LocalAudioTransportParams

load_dotenv()
logger.remove(0)
logger.add(sys.stderr, level="INFO")


async def main():
    # Create local audio transport for testing
    transport = LocalAudioTransport(
        LocalAudioTransportParams(
            audio_in_enabled=True,
            audio_out_enabled=True,
        )
    )

    # AssemblyAI U3-Pro STT
    stt = AssemblyAISTTService(
        api_key=os.getenv("ASSEMBLYAI_API_KEY"),
        connection_params=AssemblyAIConnectionParams(
            speech_model="u3-rt-pro",
            min_turn_silence=100,
            format_turns=True,
        ),
        vad_force_turn_endpoint=True,  # Pipecat mode
    )

    # Text-to-Speech
    tts = RimeTTSService(
        api_key=os.getenv("RIME_API_KEY"),
        voice_id="rex",
        model="mistv2",
    )

    # LLM
    llm = CerebrasLLMService(
        api_key=os.getenv("CEREBRAS_API_KEY"),
        model="llama-3.3-70b",
    )

    # Conversation context
    messages = [
        {
            "role": "system",
            "content": "You are a helpful voice assistant. Keep responses brief (1-2 sentences).",
        },
    ]
    context = LLMContext(messages)

    # Aggregators with VAD for Pipecat mode
    user_aggregator, assistant_aggregator = LLMContextAggregatorPair(
        context,
        user_params=LLMUserAggregatorParams(
            vad_analyzer=SileroVADAnalyzer()  # Required for Pipecat mode
        ),
    )

    # Pipeline
    pipeline = Pipeline(
        [
            transport.input(),
            stt,
            user_aggregator,
            llm,
            tts,
            transport.output(),
            assistant_aggregator,
        ]
    )

    # Task
    task = PipelineTask(
        pipeline,
        params=PipelineParams(
            enable_metrics=True,
            enable_usage_metrics=True,
        ),
    )

    # Start conversation
    await task.queue_frames([LLMRunFrame()])

    # Run
    runner = PipelineRunner()
    await runner.run(task)


if __name__ == "__main__":
    asyncio.run(main())
```

## Prompt engineering

U3-Pro supports custom transcription instructions via the `prompt` parameter:

```python
stt = AssemblyAISTTService(
    api_key=os.getenv("ASSEMBLYAI_API_KEY"),
    connection_params=AssemblyAIConnectionParams(
        speech_model="u3-rt-pro",
        min_turn_silence=100,
        format_turns=True,
        prompt="""Short utterance. Context: Transcribe short utterances such as yes or no answers.
Mandatory: Preserve linguistic speech patterns including disfluencies, filler words, hesitations,
repetitions, stutters, false starts, and colloquialisms in the spoken language.
Punctuation rules: 1) Always include punctuation in output. 2) Use period/question mark ONLY for
complete sentences. 3) Use comma for mid-sentence pauses. 4) Use no punctuation for incomplete
trailing speech.""",
    ),
)
```

### Prompt engineering tips

- **Specify audio context**: Accent, domain, expected utterance length
- **Call out problem words**: Explicitly mention frequently misrecognized terms
- **Define punctuation rules**: Helps downstream LLM processing
- **Preserve speech patterns**: Keep disfluencies for natural interactions

## Keyterms boosting

Improve recognition of specific words or names:

```python
stt = AssemblyAISTTService(
    api_key=os.getenv("ASSEMBLYAI_API_KEY"),
    connection_params=AssemblyAIConnectionParams(
        speech_model="u3-rt-pro",
        min_turn_silence=100,
        format_turns=True,
        keyterms_prompt=["Xiomara", "Saoirse", "Pipecat", "AssemblyAI"],
    ),
)
```

Alternatively, boost terms within a prompt:

```python
word_boost = ["AssemblyAI", "Pipecat", "async", "webhook"]
prompt = f"""Your transcription instructions here.
Make sure to boost the words {', '.join(word_boost)} in the audio."""

stt = AssemblyAISTTService(
    api_key=os.getenv("ASSEMBLYAI_API_KEY"),
    connection_params=AssemblyAIConnectionParams(
        speech_model="u3-rt-pro",
        prompt=prompt,
    ),
)
```

## Dynamic parameter updates

Change configuration mid-conversation without reconnection:

```python
from pipecat.frames.frames import STTUpdateSettingsFrame
from pipecat.services.assemblyai.stt import AssemblyAISTTSettings
from pipecat.services.assemblyai.models import AssemblyAIConnectionParams

# Update keyterms during conversation
await task.queue_frame(
    STTUpdateSettingsFrame(
        delta=AssemblyAISTTSettings(
            connection_params=AssemblyAIConnectionParams(
                keyterms_prompt=["NewName", "NewCompany"]
            )
        )
    )
)

# Update prompt
await task.queue_frame(
    STTUpdateSettingsFrame(
        delta=AssemblyAISTTSettings(
            connection_params=AssemblyAIConnectionParams(
                prompt="Updated transcription instructions"
            )
        )
    )
)

# Update silence thresholds
await task.queue_frame(
    STTUpdateSettingsFrame(
        delta=AssemblyAISTTSettings(
            connection_params=AssemblyAIConnectionParams(
                min_turn_silence=200,
                max_turn_silence=3000,  # Only respected in AssemblyAI's built-in turn detection (STT mode)
            )
        )
    )
)
```

## Speaker diarization

Identify different speakers in multi-party conversations.

### Basic diarization

```python
stt = AssemblyAISTTService(
    api_key=os.getenv("ASSEMBLYAI_API_KEY"),
    connection_params=AssemblyAIConnectionParams(
        speech_model="u3-rt-pro",
        speaker_labels=True,
    ),
)
```

Speaker labels (e.g., `"A"`, `"B"`, `"C"`) are included in final transcripts and logged.

### With custom formatting

Format transcripts with speaker labels for LLM context:

```python
stt = AssemblyAISTTService(
    api_key=os.getenv("ASSEMBLYAI_API_KEY"),
    connection_params=AssemblyAIConnectionParams(
        speech_model="u3-rt-pro",
        speaker_labels=True,
    ),
    speaker_format="<Speaker {speaker}>{text}</Speaker {speaker}>",
)
```

**Format options:**

| Style    | Format string                              |
| -------- | ------------------------------------------ |
| XML      | `<{speaker}>{text}</{speaker}>`            |
| Markdown | `**{speaker}**: {text}`                    |
| Bracket  | `[{speaker}] {text}`                       |

## Language detection

Enable automatic language detection (for multilingual models):

```python
stt = AssemblyAISTTService(
    api_key=os.getenv("ASSEMBLYAI_API_KEY"),
    connection_params=AssemblyAIConnectionParams(
        speech_model="universal-streaming-multilingual",
        language_detection=True,
    ),
)
```

Language codes and confidence scores are included in transcription frames. The service automatically uses detected languages with confidence >= 0.7.

## Parameters reference

### U3-Pro specific parameters

<ParamField path="speech_model" type="str" default="universal-streaming-english">
  Set to `"u3-rt-pro"` to use Universal-3 Pro.
</ParamField>

<ParamField
  path="min_turn_silence"
  type="int"
  default="400"
>
  Milliseconds of silence before ending a turn when model is confident. Set to
  `100` for best latency. (Formerly `min_end_of_turn_silence_when_confident`,
  which is deprecated but still supported with a warning.)
</ParamField>

<ParamField path="max_turn_silence" type="int" default="2000">
  Maximum silence before forced turn end. Auto-synced in Pipecat mode; respected
  in AssemblyAI's built-in turn detection (STT mode).
</ParamField>

<ParamField path="format_turns" type="bool" default="False">
  Enable turn formatting. Recommended `True` — no added latency with U3-Pro.
</ParamField>

<ParamField path="prompt" type="str">
  Custom transcription instructions. Cannot be used with `keyterms_prompt`.
</ParamField>

<ParamField path="keyterms_prompt" type="list[str]">
  List of terms to boost recognition for. Cannot be used with `prompt`.
</ParamField>

<ParamField path="speaker_labels" type="bool" default="False">
  Enable speaker diarization.
</ParamField>

### General parameters

<ParamField path="api_key" type="str" required>
  Your AssemblyAI API key.
</ParamField>

<ParamField path="vad_force_turn_endpoint" type="bool" default="True">
  `True` for Pipecat mode; `False` for AssemblyAI's built-in turn detection (STT mode).
</ParamField>

<ParamField path="speaker_format" type="str">
  Template string for formatting speaker labels (e.g.,
  `"[{speaker}] {text}"`).
</ParamField>

<ParamField path="language_detection" type="bool" default="False">
  Enable automatic language detection.
</ParamField>

## Troubleshooting

| Issue                                    | Cause                                                    | Solution                                                                          |
| ---------------------------------------- | -------------------------------------------------------- | --------------------------------------------------------------------------------- |
| "Are you there?" moments                | High end-of-turn latency                                 | Set `min_turn_silence=100`                                                        |
| Over-segmentation (cutting off user)     | `min_turn_silence` too low                               | Increase to `200`–`300`                                                           |
| Missing VAD warning in Pipecat mode      | No VAD analyzer in user aggregator                       | Add `LLMUserAggregatorParams(vad_analyzer=SileroVADAnalyzer())`                   |
| AssemblyAI's built-in turn detection not working | Using non-`u3-rt-pro` model                       | Only `u3-rt-pro` supports AssemblyAI's built-in turn detection (STT mode)         |
| `max_turn_silence` warning              | Setting `max_turn_silence` in Pipecat mode               | Expected — it's auto-synced (or switch to AssemblyAI's built-in turn detection)   |
| Dynamic updates not working             | Using old dict format                                    | Use `STTUpdateSettingsFrame(delta=AssemblyAISTTSettings(...))`                     |
| Speaker labels always None              | Missing `speaker_labels=True`                            | Enable in `connection_params`                                                     |
| Prompt + keyterms error                 | Both set simultaneously                                  | Choose one or the other                                                           |

### Known limitations (Alpha)

- **AssemblyAI's built-in turn detection (STT mode) is experimental**: Pipecat mode is recommended for most use cases
- **No partial transcripts**: U3-Pro only returns final transcripts
- **Dynamic prompt updates**: Supported but requires AssemblyAI API support (in progress)
- **Not yet production-grade**: Alpha status — contact AssemblyAI before deploying to production

## Migration from standard AssemblyAI STT

If migrating from `universal-streaming` models:

| Change                                              | From                                 | To                                                           |
| --------------------------------------------------- | ------------------------------------ | ------------------------------------------------------------ |
| Model                                               | Default (`universal-streaming-english`) | `speech_model="u3-rt-pro"`                                |
| Turn detection                                      | Automatic                            | Choose mode: Pipecat (default) or AssemblyAI's built-in (STT mode) |
| VAD requirement                                     | Optional                             | **Required** in Pipecat mode                                 |
| `min_turn_silence`                                  | Default (`400`)                      | `100` recommended                                            |
| Parameter naming                                    | N/A                                  | `min_end_of_turn_silence_when_confident` deprecated, use `min_turn_silence` |
| Prompting                                           | Not available                        | Available via `prompt` parameter                             |
| Keyterms                                            | Not available                        | Available via `keyterms_prompt`                               |
| Speaker diarization                                 | Not available                        | Available via `speaker_labels=True`                           |
| Dynamic updates                                     | Not available                        | Supported via `STTUpdateSettingsFrame`                        |
| `max_turn_silence` in Pipecat mode                  | Configurable                         | Auto-synchronized (or switch to AssemblyAI's built-in turn detection) |

## Full example with all features

```python
import asyncio
import os

from dotenv import load_dotenv

from pipecat.audio.vad.silero import SileroVADAnalyzer
from pipecat.frames.frames import LLMRunFrame, STTUpdateSettingsFrame
from pipecat.pipeline.pipeline import Pipeline
from pipecat.pipeline.runner import PipelineRunner
from pipecat.pipeline.task import PipelineParams, PipelineTask
from pipecat.processors.aggregators.llm_context import LLMContext
from pipecat.processors.aggregators.llm_response_universal import (
    LLMContextAggregatorPair,
    LLMUserAggregatorParams,
)
from pipecat.services.assemblyai.models import AssemblyAIConnectionParams
from pipecat.services.assemblyai.stt import AssemblyAISTTService, AssemblyAISTTSettings
from pipecat.services.rime.tts import RimeTTSService
from pipecat.services.cerebras.llm import CerebrasLLMService
from pipecat.transports.local.audio import LocalAudioTransport, LocalAudioTransportParams

load_dotenv()


async def main():
    transport = LocalAudioTransport(
        LocalAudioTransportParams(
            audio_in_enabled=True,
            audio_out_enabled=True,
        )
    )

    # U3-Pro with all features enabled
    stt = AssemblyAISTTService(
        api_key=os.getenv("ASSEMBLYAI_API_KEY"),
        connection_params=AssemblyAIConnectionParams(
            speech_model="u3-rt-pro",
            min_turn_silence=100,
            format_turns=True,
            speaker_labels=True,                                    # Enable diarization
            keyterms_prompt=["Pipecat", "AssemblyAI", "u3-rt-pro"], # Boost key terms
        ),
        vad_force_turn_endpoint=True,          # Pipecat mode
        speaker_format="[{speaker}] {text}",   # Format speaker labels
    )

    tts = RimeTTSService(
        api_key=os.getenv("RIME_API_KEY"),
        voice_id="rex",
        model="mistv2",
    )

    llm = CerebrasLLMService(
        api_key=os.getenv("CEREBRAS_API_KEY"),
        model="llama-3.3-70b",
    )

    messages = [
        {
            "role": "system",
            "content": "You are a helpful voice assistant. Keep responses brief.",
        },
    ]
    context = LLMContext(messages)

    user_aggregator, assistant_aggregator = LLMContextAggregatorPair(
        context,
        user_params=LLMUserAggregatorParams(
            vad_analyzer=SileroVADAnalyzer()
        ),
    )

    pipeline = Pipeline(
        [
            transport.input(),
            stt,
            user_aggregator,
            llm,
            tts,
            transport.output(),
            assistant_aggregator,
        ]
    )

    task = PipelineTask(
        pipeline,
        params=PipelineParams(
            enable_metrics=True,
            enable_usage_metrics=True,
        ),
    )

    # Example: dynamically update keyterms after 10 seconds
    async def update_keyterms():
        await asyncio.sleep(10)
        await task.queue_frame(
            STTUpdateSettingsFrame(
                delta=AssemblyAISTTSettings(
                    connection_params=AssemblyAIConnectionParams(
                        keyterms_prompt=["NewTerm", "UpdatedName"]
                    )
                )
            )
        )

    asyncio.create_task(update_keyterms())

    await task.queue_frames([LLMRunFrame()])

    runner = PipelineRunner()
    await runner.run(task)


if __name__ == "__main__":
    asyncio.run(main())
```
