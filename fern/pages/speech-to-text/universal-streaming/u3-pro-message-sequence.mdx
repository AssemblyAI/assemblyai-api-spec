Universal-3 Pro handles message sequences differently from [Universal Streaming](/docs/streaming/message-sequence). Instead of emitting word-by-word partial transcripts as audio is processed, Universal-3 Pro produces complete, formatted transcripts after periods of silence. Key differences include:

- **No word-by-word partials** — transcripts are only emitted during silence periods, not on every audio frame.
- **Formatting is built in** — `turn_is_formatted` is always `true` on end-of-turn transcripts. There is no separate formatting step.
- **Punctuation-based turn detection** — turns end when terminal punctuation (`.` `?` `!`) is detected, not based on a confidence threshold.
- **`end_of_turn_confidence` is always `1`** when triggered by terminal punctuation.

For this example, we walk through a user saying:
`Hi. My name is Sonny.`

## Session initialization

When the session begins, you receive a `Begin` message with the session ID and expiration time.

```json
{
  "type": "Begin",
  "id": "4015768a-b856-46bf-afa0-e03ab845d89c",
  "expires_at": 1772568751
}
```

## First turn

### Speech detected

Before any Turn messages are sent, the server sends a `SpeechStarted` message indicating that speech has been detected. The `timestamp` field indicates when the speech was detected, in milliseconds relative to the beginning of the audio stream.

```json
{
  "type": "SpeechStarted",
  "timestamp": 2336
}
```

### End of turn (Final transcript)

After the speaker says "Hi." and pauses, Universal-3 Pro transcribes the buffered audio and detects terminal punctuation (`.`). The turn ends immediately with a fully formatted final transcript.

Notice that:
- `end_of_turn` is `true` — the turn has ended.
- `turn_is_formatted` is `true` — formatting is always built in for Universal-3 Pro.
- `end_of_turn_confidence` is `1` — terminal punctuation triggered the end of turn.
- All words have `word_is_final: true` — this is a final transcript.
- The `utterance` field contains the finalized utterance text.

```json
{
  "turn_order": 0,
  "turn_is_formatted": true,
  "end_of_turn": true,
  "transcript": "Hi.",
  "end_of_turn_confidence": 1,
  "words": [
    {
      "start": 2336,
      "end": 3071,
      "text": "Hi.",
      "confidence": 0.645748,
      "word_is_final": true
    }
  ],
  "utterance": "Hi.",
  "type": "Turn"
}
```

Unlike Universal Streaming, there is no separate formatting message. The end-of-turn transcript is always formatted.

## Second turn

### Speech detected

When the user begins speaking again, another `SpeechStarted` message is sent.

```json
{
  "type": "SpeechStarted",
  "timestamp": 3136
}
```

### End of turn (Final transcript)

The speaker says "My name is Sonny." and pauses. Universal-3 Pro transcribes the full utterance and detects terminal punctuation, ending the turn with a formatted final transcript.

```json
{
  "turn_order": 1,
  "turn_is_formatted": true,
  "end_of_turn": true,
  "transcript": "My name is Sonny.",
  "end_of_turn_confidence": 1,
  "words": [
    {
      "start": 3136,
      "end": 3337,
      "text": "My",
      "confidence": 0.950153,
      "word_is_final": true
    },
    {
      "start": 3357,
      "end": 3759,
      "text": "name",
      "confidence": 0.998939,
      "word_is_final": true
    },
    {
      "start": 3779,
      "end": 3980,
      "text": "is",
      "confidence": 0.999285,
      "word_is_final": true
    },
    {
      "start": 4000,
      "end": 4544,
      "text": "Sonny.",
      "confidence": 0.446247,
      "word_is_final": true
    }
  ],
  "utterance": "My name is Sonny.",
  "type": "Turn"
}
```

## Session termination

When the session ends, a `Termination` message is sent with the total audio and session durations.

```json
{
  "type": "Termination",
  "audio_duration_seconds": 17,
  "session_duration_seconds": 17
}
```

## Partials (mid-turn transcripts)

In the example above, both turns ended immediately because terminal punctuation was detected on the first silence check. However, if the speaker pauses mid-sentence without terminal punctuation, Universal-3 Pro emits a **partial transcript** before the turn ends.

For example, if a user says "Yeah my credit card number is" and pauses, a partial is emitted:

```json
{
  "turn_order": 0,
  "turn_is_formatted": true,
  "end_of_turn": false,
  "transcript": "Yeah my credit card number is--",
  "end_of_turn_confidence": 0,
  "words": [
    {
      "start": 1000,
      "end": 1200,
      "text": "Yeah",
      "confidence": 0.95,
      "word_is_final": false
    }
  ],
  "utterance": "",
  "type": "Turn"
}
```

Key differences from end-of-turn transcripts:

- `end_of_turn` is `false` — the turn has not ended.
- All words have `word_is_final: false` — the transcript may be revised in the final end-of-turn message.
- Each silence period produces **at most one partial**.

When the speaker eventually finishes with terminal punctuation, the final end-of-turn transcript includes the complete, refined text with all words marked `word_is_final: true`.

For more details on how partials work and how to tune turn detection timing, see [Turn Detection and Partials](/docs/streaming/universal-3-pro/turn-detection-and-partials).

## Comparison with Universal Streaming

| Behavior | Universal-3 Pro | Universal Streaming |
| --- | --- | --- |
| **Partial frequency** | Only during silence periods | Every audio frame (word-by-word) |
| **Formatting** | Built in to every end-of-turn transcript | Separate `turn_is_formatted` message after end of turn |
| **Turn detection** | Punctuation-based (`min_turn_silence` / `max_turn_silence`) | Confidence-based (`end_of_turn_confidence_threshold`) |
| **`end_of_turn_confidence`** | Always `1` when triggered by punctuation | Varies based on model confidence |
| **Words in partials** | All `word_is_final: false` | Mix of `true` and `false` as words are finalized incrementally |

For the Universal Streaming message sequence, see [Message Sequence](/docs/streaming/message-sequence).
