title: "Audio Tags"
---

Audio tags allow you to capture non-speech events in your audio, such as laughter, silence, noise, coughs, and sighs. When enabled, the model will detect and tag these sounds in your transcript.

## How audio tags work

When there is a pause in the speech where a non-speech sound is detected, audio tags will be added to the transcript like `[NOISE]`, `[MUSIC]`, `[APPLAUSE]`, etc.

## Enabling audio tags

To enable audio tags, include the following instruction in your prompt:

```text
Tag sounds: [laughter], [silence], [noise], [cough], [sigh].
```

## Supported audio tags

The model can detect and tag the following non-speech events:

| Tag | Description |
|-----|-------------|
| `[NOISE]` | General background noise |
| `[MUSIC]` | Music playing |
| `[APPLAUSE]` | Clapping or applause |
| `[LAUGHTER]` | Laughter |
| `[SILENCE]` | Extended silence |
| `[COUGH]` | Coughing |
| `[SIGH]` | Sighing |

## Example usage

<CodeBlocks>

```python title="Python SDK"
import assemblyai as aai

aai.settings.api_key = "<YOUR_API_KEY>"

audio_file = "https://assembly.ai/wildfires.mp3"

config = aai.TranscriptionConfig(
    speech_models=["universal-3-pro"],
    prompt="Tag sounds: [laughter], [silence], [noise], [cough], [sigh]."
)

transcript = aai.Transcriber(config=config).transcribe(audio_file)

if transcript.status == "error":
    raise RuntimeError(f"Transcription failed: {transcript.error}")

print(transcript.text)
```

```javascript title="JavaScript SDK"
import { AssemblyAI } from "assemblyai";

const client = new AssemblyAI({
  apiKey: "<YOUR_API_KEY>",
});

const audioFile = "https://assembly.ai/wildfires.mp3";

const params = {
  audio: audioFile,
  speech_models: ["universal-3-pro"],
  prompt: "Tag sounds: [laughter], [silence], [noise], [cough], [sigh]."
};

const run = async () => {
  const transcript = await client.transcripts.transcribe(params);
  console.log(transcript.text);
};

run();
```

## Best practices

- Audio tags work best with Universal-3-Pro model
- Combine audio tags with other prompt instructions for comprehensive transcription control
- Audio tags are most useful for content analysis, accessibility features, and detailed transcription requirements

For more information on prompt engineering and customizing transcription output, see [Natural language prompting](/docs/speech-to-text/pre-recorded-audio/natural-language-prompting).
